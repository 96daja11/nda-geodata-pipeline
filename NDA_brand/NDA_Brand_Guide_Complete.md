## 1. FÖRETAGSNAMNET

### Stavning
Företagsnamnet är alltid **två ord**: `North Drone Analytics`

| ✅ Korrekt | ❌ Fel |
|---|---|
| North Drone Analytics | NorthDroneAnalytics |
| North Drone **Analytics** *(med accentfärg)* | North-Drone Analytics |
| | north drone analytics |
| | NDA *(förkortning i löptext)* |

### Kulörregler för namnet

Principen: "North Drone" sätts i neutral kulör, "Analytics" markeras alltid med primär accentfärg.
Denna uppdelning understryker att vi är ett **dataanalysföretag** — inte en drönaroperatör.

| Bakgrund | "North Drone" | "Analytics" | Hex-koder |
|---|---|---|---|
| Mörk (`#08152A` / `#0F2240`) | Vit | Blue light | `#FFFFFF` + `#4B8AF4` |
| Ljus (`#FFFFFF` / `#F7F9FC`) | Navy | Blue | `#08152A` + `#1D55D4` |
| Accentblå (`#1D55D4`) | Vit | Blue pale | `#FFFFFF` + `#D6E6FF` |

**HTML-exempel — mörk bakgrund:**
```html
<span style="color:#FFFFFF; font-family:'DM Sans', sans-serif; font-weight:700;">North Drone</span>
<span style="color:#4B8AF4; font-family:'DM Sans', sans-serif; font-weight:700;">Analytics</span>
```

## 2. KULÖRPALETTEN

### CSS-variabler
Använd **alltid** dessa CSS-variabler. Hårdkoda aldrig hex-värden direkt i komponenter.

```css
:root {
  /* ── Primärpaletten ── */

  --navy-mid:      #0F2240;  /* Sekundär bakgrund, band, kortbakgrund på mörk yta */
  --blue:          #1D55D4;  /* Primär accentfärg — CTA-knappar, aktiva element */
  --blue-light:    #4B8AF4;  /* Länkar, hover-states, ikoner, sekundära accenter */
  --blue-pale:     #D6E6FF;  /* Kortbakgrunder, highlights, subtila markeringar */
  --blue-border:   #A8C4F8;  /* Kanter och ramar på blå kort */

  /* ── Teal — Vatten & Avlopp ── */
  --teal:          #0E9F8E;  /* Rubrikfärg, ikonkuölr, aktiva element */
  --teal-pale:     #C4EAE6;  /* Kortbakgrund */
  --teal-border:   #7DCDC6;  /* Kortram */

  /* ── Grön — Mätning & GIS ── */

  /* ── Neutrala ── */

  --muted:         #667388;  /* Sekundär text, bildtexter, metadata */

```

### Accentpaletternas användning
Varje accentpalett har tre nivåer. Använd dem alltid i kombination:
- **Solid** → rubriktext, ikonkuölr, aktiva etiketter
- **Pale** → kortbakgrund
- **Border** → kortram/separator

```css
/* Exempel — ett tjänstekort för Vatten & Avlopp */
.service-card-va {
  background: var(--teal-pale);       /* #C4EAE6 */
  border: 1px solid var(--teal-border); /* #7DCDC6 */
}
.service-card-va .card-title {
  color: var(--teal);                 /* #0E9F8E */
}
```

## 4. LOGOTYPER — FULLSTÄNDIG SPECIFIKATION

### Teknisk bakgrund
- Alla logotyper är SVG-filer med viewBox `0 0 2000 1846`
- Logotypens grafik är placerad i mitten av detta canvas
- Sökvägar använder SVG relativa `c`-kommandon (cubic bezier)
- Logotyperna ska **alltid** bäddas in som inline SVG — inte som `<img src="...">` — för korrekt skalning och kulörkontroll

| 48 × 48 px | 8 px | Sidhuvud, mellanstorlek |
| 32 × 32 px | 5 px | Favikon |
| 20 × 20 px | 3 px | Minimum digital |

## 6. LOGOTYPREGLER

### Vad som alltid gäller
- Skyddszon runt logotypen = minst logotypikonens höjd på alla sidor
- Skala alltid proportionellt — ändra aldrig bara bredd eller bara höjd
- Signalvågorna (blå accent) ska alltid vara synliga och ej beskurna
- Använd alltid inline SVG — aldrig `<img src="logo.svg">`

### Förbjudet
| Förbjudet | Anledning |
|---|---|
| Placera på kulörglada/fotografiska bakgrunder | Logotypen försvinner |
| Ändra kulörerna på paths | Bryter varumärkets identitet |
| Lägga skuggor, glow eller outlines | Ser oprofessionellt ut |
| Rotera eller vinkla logotypen | Aldrig tillåtet |
| Använda `<img>` istället för inline SVG | Förlorar kulörkontroll |
| Använda `viewBox="0 0 2000 1846"` för Small-logotypen | Ikonen hamnar i fel position |

## 10. IKONOGRAFI — LUCIDE ICONS

### Bibliotek
**Lucide Icons** används genomgående för alla UI-ikoner på hemsidan.
- Webbplats: https://lucide.dev
- Installation: `npm install lucide-react` (React) eller via CDN
- **Blanda aldrig Lucide med andra ikonbibliotek** (t.ex. Heroicons, FontAwesome, Material Icons) på samma sida

### Stil — alltid outline, aldrig filled

| Egenskap | Värde | Kommentar |
|---|---|---|
| Stil | Outline (stroke) | Aldrig filled |
| `stroke-width` | `1.5` | Default — ändras ej |
| `stroke-linecap` | `round` | Mjuka avslutningar |
| `stroke-linejoin` | `round` | Mjuka hörn |
| `fill` | `none` | **Alltid none — undantag finns ej** |

**Korrekt SVG-attribut:**
```html
<svg
  viewBox="0 0 24 24"
  fill="none"
  stroke="currentColor"
  stroke-width="1.5"
  stroke-linecap="round"
  stroke-linejoin="round"
  width="24"
  height="24"
>
  <!-- path här -->
</svg>
```

**Fel — aldrig göra:**
```html
<!-- ❌ fill används aldrig -->
<svg fill="white" stroke="none">...</svg>

<!-- ❌ stroke-width ska inte ändras från 1.5 -->
<svg stroke-width="2">...</svg>

<!-- ❌ aldrig blanda ikonbibliotek -->
<i class="fa fa-home"></i> <!-- FontAwesome förbjudet -->
```

### Storlekar per kontext

| Kontext | Storlek | Användning |
|---|---|---|
| Inline / text | `16px` | Ikon bredvid text i löptext |
| UI-element | `20px` | Knappar, inputs, navigationslänkar |
| Tjänstekort | `24px` | Kortrubriker, feature-ikoner |
| Hero / feature | `32–40px` | Stora featuresektioner, landingpage |
| Ikonknapp | `20px i 40px wrap` | Klickbara ikonknappar |

## 11. FORMSPRÅK — RUNDADE HÖRN, SKUGGOR & SPACING

### Principen
Alla element på hemsidan har rundade hörn. **Skarpa raka kanter används aldrig.**
Skuggor ger djup och hierarki — kortare mjukare skuggor för element i vila, mer upplyft vid hover.

### Border-radius skala

| Token | Värde | Används för |
|---|---|---|
| Extra liten | `border-radius: 4px` | Taggar, badges, etiketter |
| Liten | `border-radius: 8px` | Knappar, inputs, small chips |
| Medium | `border-radius: 12px` | Tjänstekort, sektioner |
| Stor | `border-radius: 16px` | Bildrutor, hero-kort |
| Pill | `border-radius: 999px` | Primärknappar, pill-etiketter |
| Rund | `border-radius: 50%` | Avatarer, ikonknappar |

```css
:root {
  --radius-xs:   4px;
  --radius-sm:   8px;
  --radius-md:   12px;
  --radius-lg:   16px;
  --radius-pill: 999px;
  --radius-full: 50%;
}
```

### Skuggor

| Token | CSS-värde | Används för |
|---|---|---|
| Subtil | `0 1px 3px rgba(8,21,42,0.08)` | Kort i vila |
| Standard | `0 4px 16px rgba(8,21,42,0.10)` | Tjänstekort, sektioner |
| Upplyft | `0 8px 32px rgba(8,21,42,0.14)` | Hover-state på kort |
| Modal | `0 20px 60px rgba(8,21,42,0.18)` | Modaler, dropdowns |

```css
:root {
  --shadow-sm:    0 1px 3px rgba(8,21,42,0.08);
  --shadow-md:    0 4px 16px rgba(8,21,42,0.10);
  --shadow-lg:    0 8px 32px rgba(8,21,42,0.14);
  --shadow-modal: 0 20px 60px rgba(8,21,42,0.18);
}
```

### Spacing
Använd multiples av 4px som basenhet för all spacing.

```css
:root {
  --space-1:  4px;
  --space-2:  8px;
  --space-3:  12px;
  --space-4:  16px;
  --space-5:  20px;
  --space-6:  24px;
  --space-8:  32px;
  --space-10: 40px;
  --space-12: 48px;
  --space-16: 64px;
}
```

## 12. TJÄNSTEKORT — KOMPONENTER & STATES

### Kortstruktur
Varje tjänstekort består av fyra lager uppifrån och ned:
1. **Bildyta** — stilren drönarfoto, 16:9 eller 4:3, med `border-radius: 12px 12px 0 0`
2. **Ikonwrapper** — 48×48px rund/kvadratisk yta i tjänstens solid-kulör
3. **Kortbody** — rubrik, brödtext, "Läs mer"-länk
4. **Kortram** — vit bakgrund, `border-radius: 12px`, skugga

### Default state
```css
.service-card {
  background: #FFFFFF;
  border-radius: 12px;
  border: 1px solid var(--border);          /* #E4EAF2 */
  box-shadow: var(--shadow-sm);             /* 0 1px 3px rgba(8,21,42,0.08) */
  overflow: hidden;
  transition: box-shadow 0.2s ease, border-color 0.2s ease, transform 0.2s ease;
}
```

### Hover state
Vid hover lyfts kortet med ökad skugga och en färgad border som matchar **tjänstens solid-kulör**.

```css
.service-card:hover {
  box-shadow: var(--shadow-lg);             /* 0 8px 32px rgba(8,21,42,0.14) */
  border-color: var(--service-color);       /* Se tabell nedan */
  border-width: 2px;
  transform: translateY(-2px);             /* Subtil lyfteffekt */
}
```

### Hover-borderfärg per tjänsteområde

| Tjänsteområde | Hover-borderfärg | CSS-variabel |
|---|---|---|
| Vatten & Avlopp | `#0E9F8E` | `var(--teal)` |
| Mätning & GIS | `#18A556` | `var(--green)` |
| Fastighet & Förvaltning | `#D97706` | `var(--amber)` |
| Leverans & Analys | `#7038E8` | `var(--purple)` |
| Generisk / primär | `#1D55D4` | `var(--blue)` |

**Regel:** Hover-borderfärgen ska alltid matcha ikonbakgrundens solid-kulör på samma kort.
Blanda aldrig — ett VA-kort får aldrig amber-border vid hover.

### Ikonwrapper i kortets header

```html
  margin: 16px 16px 8px;
">
  <svg viewBox="0 0 24 24" fill="none" stroke="white"
       stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"
```

## 13. KNAPPAR — STATES & VARIANTER

### Principer
- Alla knappar har `border-radius: 999px` (pill-form)
- Tre varianter: **Primär** (fylld), **Ghost** (outline på mörk bakgrund), **Outline** (outline på ljus bakgrund)
- Alla states har en `transition` på 0.15–0.2s för mjuk feedback
- Active-state (klick): `transform: scale(0.97)` — subtilt intryck

### Primär knapp (Filled)

```css
.btn-primary {
  background: var(--blue);        /* #1D55D4 */
  color: #FFFFFF;
  border: none;
  border-radius: 999px;
  padding: 12px 24px;
  font-family: "DM Sans", sans-serif;
  font-weight: 700;
  font-size: 15px;
  cursor: pointer;
  transition: background 0.15s ease, box-shadow 0.15s ease, transform 0.15s ease;
}
.btn-primary:hover {
  background: var(--blue-light);  /* #4B8AF4 — ljusare vid hover */
  box-shadow: 0 4px 12px rgba(29,85,212,0.35);
}
.btn-primary:active {
  transform: scale(0.97);
}
```

### Ghost-knapp (Outline på mörk bakgrund)

Används alltid mot mörk bakgrund (`--navy`, `--navy-mid`).
**Hover-effekt: knappen fyller sig med vit bakgrund och text blir navy.**

```css
.btn-ghost {
  background: transparent;
  color: #FFFFFF;
  border: 1.5px solid rgba(255,255,255,0.6);
  border-radius: 999px;
  padding: 12px 24px;
  font-family: "DM Sans", sans-serif;
  font-weight: 600;
  font-size: 15px;
  cursor: pointer;
  transition: background 0.15s ease, border-color 0.15s ease, color 0.15s ease;
}
.btn-ghost:hover {
  background: #FFFFFF;            /* Fylls med vitt */
  color: var(--navy);             /* Text blir navy */
  border-color: #FFFFFF;
}
.btn-ghost:active {
  transform: scale(0.97);
}
```

### Outline-knapp (på ljus bakgrund)

```css
.btn-outline {
  background: transparent;
  color: var(--navy);
  border: 1.5px solid var(--navy);
  border-radius: 999px;
  padding: 12px 24px;
  font-family: "DM Sans", sans-serif;
  font-weight: 600;
  font-size: 15px;
  cursor: pointer;
  transition: background 0.15s ease, color 0.15s ease;
}
.btn-outline:hover {
  background: var(--navy);        /* Fylls med navy */
  color: #FFFFFF;                 /* Text blir vit */
}
.btn-outline:active {
  transform: scale(0.97);
}
```

### Sammanfattning — alla states

| Variant | Default | Hover | Active |
|---|---|---|---|
| Primär | Blå fylld, vit text | Ljusblå fylld + skugga | scale(0.97) |
| Ghost (mörk bg) | Transparent, vit border/text | Vit fylld, navy text | scale(0.97) |
| Outline (ljus bg) | Transparent, navy border/text | Navy fylld, vit text | scale(0.97) |

### Knappstorlekar

| Storlek | Padding | Font-size | Användning |
|---|---|---|---|
| Liten | `8px 16px` | `13px` | Sekundära åtgärder, kort |
| Standard | `12px 24px` | `15px` | Primär CTA, sektioner |
| Stor | `16px 32px` | `17px` | Hero-sektion |

## 14. BILDSTIL & FOTOGRAFI

### Principen
Alla bilder ska förmedla **precision, teknisk kompetens och professionalism**.
Bilderna är alltid rena, välkomponerade och tekniskt korrekta.

### Bildkrav — alltid

| Krav | Detalj |
|---|---|
| Drönare | Professionella modeller — DJI Matrice, Autel EVO II eller liknande |
| Komposition | Skarp fokus, ren bakgrund, välbalanserad exponering |
| Perspektiv | Fågelperspektiv på infrastruktur föredras |
| Termiska bilder | Blå/mörkblå palett — matchar varumärkets kulörer |
| Hörn | Alltid `border-radius: 12–16px` på bildrutor |
| Overlay | Lätt mörkare gradient (`linear-gradient`) om text placeras ovanpå |
| Format | 16:9 eller 4:3 för kortbilder · 1:1 för thumbnails |

### Förbjudet

| Förbjudet | Anledning |
|---|---|
| Generiska stockfoton av "drönare i solnedgång" | Ser amatörmässigt ut |
| Övermättade, HDR-bearbetade eller Instagram-filtrerade bilder | Bryter varumärkets stilrena känsla |
| Konsumentdrönare (DJI Mini, Phantom) | Signalerar fel kompetensnivå |
| Suddiga, lågupplösta eller skeva bilder | Undermålig kvalitet |
| Bilder med skarpa raka hörn (ingen border-radius) | Bryter formspråket |
| Mättade orangea/röda färger | Krockar med varumärkets blå palett |
| Foton av identifierbara personer utan samtycke | GDPR |

### Bildtyper som fungerar

```
✅ Professionell drönare mot molnig himmel eller industriell miljö
✅ Fågelperspektiv på VA-ledningar, byggnader, markstrukturer
✅ Termiska bilder med tydliga temperaturgradienter i blå/mörkblå palett
✅ Punktmoln, ortofoton och 3D-visualiseringar
✅ Närbild på professionell kamera/sensor monterad på drönare
✅ Dashboard/data-visualiseringar i varumärkets färger
```
